CRITICAL ISSUES (2)
================================================================================

CRITICAL #1: RunManager Initialization Broken
────────────────────────────────────────────────────────────────────────────
File:     autoload/run_manager.gd (lines 53, 69)
Problem:  Phase 2 removed hand_size from Run and RunState.start_run()
          signature, but RunManager wasn't updated.

Line 53:  run_state.start_run(run.plays_per_round, run.hand_size, run.bag_config)
          should be: run_state.start_run(run.plays_per_round, run.bag_config)

Line 69:  print references run_state.hand_size which no longer exists

Impact:   Game cannot start - TypeError on run initialization
Fix Time: 2 minutes


CRITICAL #2: Signal Cleanup Race Condition in HandFanLayout
────────────────────────────────────────────────────────────────────────────
File:     scripts/animation/hand/hand_fan_layout.gd (_unregister_tile method)
Problem:  Signals disconnected AFTER validity check, so freed tiles never
          get signal cleanup. Signals remain connected to invalid object.

Line 181: if not is_instance_valid(tile): return  ← exits before disconnect
Lines 193-199: Disconnect code never reached for freed tiles

Impact:   Signal emissions from freed tiles, memory leaks, potential crashes
          during round transitions when hand is cleared
Fix Time: 5 minutes

IMPORTANT ISSUES (2)
================================================================================

IMPORTANT #1: Draw Button Doesn't Check Hand Full
────────────────────────────────────────────────────────────────────────────
File:     scenes/ui/main_hud/main_hud.gd (lines 233-237)
Current:  draw_button.disabled = TileBag.is_empty()
Should be: draw_button.disabled = TileBag.is_empty() or HandManager.is_hand_full()

Impact:   Players can click draw when hand is full, button silently fails
Fix Time: 1 minute


IMPORTANT #2: Hand.is_full() Method Missing
────────────────────────────────────────────────────────────────────────────
File:     scenes/hand/hand.gd
Problem:  HandManager.is_hand_full() calls _hand_ui.is_full() but Hand
          component doesn't expose this method.

Impact:   Runtime error when draw button state updates
Fix Time: 2 minutes
